# MAGISTRATYRA_TASK_SOLUTION

## Описание решения
Вероятнее всего, мы имеем дело с совсем немного предобработанными данными БКИ - их просто дедуплицировали

В качетсве решения проделана следующая работа:
- **Агрегация:** Агрегированы результаты одного клиента по нескольким кредитам - таким образом учитываем всю кредитную историю для предсказания дефолта
- **Фич-инжиниринг:** Добавлены некоторые фичи, которые могут помочь предсказанию (так понимаю) дефолта
- **Подбор гиперпараметров и обучение:** Подбирал гиперпараметры с помощью байесовской оптимизации, используя градиентный бустинг

## Проблемы с которыми пришлось столкнуться:
- Огромные объемы данных (именно они подтолкнули использовать агрегацию)
- Погружение в предметную область, для получения фичей
